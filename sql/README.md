# AppMetrica + CRM Attribution Dataset

## Задача

Для проекта были выгружены сырые данные из AppMetrica и CRM клиента:

- installs (установки);
- purchases (покупки);
- deeplinks (диплинки);
- CRM-данные.

Необходимо было объединить данные в единый датасет и корректно атрибутировать покупки к рекламным источникам.

## Логика атрибуции

- Покупке присваивается источник установки либо источник диплинка в окне 30 дней.
- Приоритетным источником является диплинк.
- Если с момента установки прошло более 30 дней — заказ относится к органическому трафику.

## Дедупликация

Дополнительно реализована дедупликация заказов, так как один и тот же лид мог одновременно попадать:

- в app-метрику;
- в web-метрику.

Для решения задачи использовался отдельный словарь дедупликации, который автоматически формировался на основе заказов, присутствующих в обеих системах.

## Особенности реализации

### 1. Атрибуция по дате покупки

В качестве основной даты атрибуции используется дата покупки.

Это позволяет корректно привязывать заказ к источнику установки или диплинка, однако требует отдельной логики расчета installs по датам.

### 2. Очистка дублей заказов

AppMetrica может отдавать дублирующиеся заказы.

Для удаления дублей использовалась оконная функция `row_number()` по id заказа.

Все записи с номером строки > 1 исключались из итогового датасета.

## Результат

В результате был сформирован итоговый датасет:

- без дубликатов;
- с корректной атрибуцией источников;
- с сохранением консистентности данных.

Например:

- сумма revenue в CRM до обработки полностью совпадает с итоговой суммой после обработки;
- каждому заказу присвоен источник в соответствии с логикой атрибуции.

Далее датасет используется:

- для сегментации рекламных источников;
- построения BI-дашбордов;
- анализа эффективности каналов;
- расчета маркетинговых метрик.
