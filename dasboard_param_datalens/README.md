## Описание проекта

Проект демонстрирует возможности Yandex DataLens по созданию динамических визуализаций с использованием параметров (Parameters) и вычисляемых полей.

Основная задача заключалась в том, чтобы реализовать универсальный dashboard, позволяющий пользователю:

- динамически изменять период агрегации данных;
- переключать отображаемые показатели;
- использовать один и тот же чарт для разных аналитических сценариев;
- минимизировать количество дублирующихся графиков и таблиц.

---

# Реализованный функционал

## Параметр периода (`scaleru`)

Создан параметр для изменения уровня агрегации данных.

Доступные значения:

- Год
- Квартал
- Месяц
- Неделя
- День

Пользователь может переключать временной масштаб прямо в dashboard без изменения настроек чарта.

---

## Параметр показателя (`metric`)

Создан параметр для переключения бизнес-показателей.

Поддерживаемые метрики:

- Визиты
- Динамика визитов, %
- Количество заказов
- Сумма заказов
- Количество заказанных SKU
- Выкупаемость
- Установки
- и другие показатели

Это позволяет использовать один чарт для отображения различных KPI.

---

# Используемые формулы

## Динамическая агрегация даты

```sql
IF [scaleru]="Год" THEN DATETRUNC([date],"year")
ELSEIF [scaleru]="Квартал" THEN DATETRUNC([date],"quarter")
ELSEIF [scaleru]="Месяц" THEN DATETRUNC([date],"month")
ELSEIF [scaleru]="Неделя" THEN DATETRUNC([date],"week")
ELSEIF [scaleru]="День" THEN DATETRUNC([date],"day")
END```

## Динамический выбор показателя:

if(
    [metric]='Визиты', SUM([visits]),
    [metric]='Динамика визитов, %', [Динамика визитов],
    [metric]='Кол-во заказов', SUM([purchases]),
    [metric]='Сумма заказов, руб', SUM([revenue]),
    [metric]='Кол-во заказанных шт товара', SUM([qty_sku_zakaza]),
    [metric]='Выкупаемость', [Выкупаемость],
    [metric]='Установки', SUM([installs]),
    null
)

## Особенности реализации

В стандартных dashboard DataLens для каждой метрики и периода часто создаются отдельные графики.

В данном проекте реализован универсальный подход:

-один чарт поддерживает разные периоды агрегации;
-один чарт поддерживает разные метрики;
-логика полностью управляется параметрами;
-пользователь самостоятельно управляет аналитикой через интерфейс dashboard.

Это позволяет значительно сократить количество визуализаций и сделать dashboard более удобным и масштабируемым.

# Результат

В результате был создан динамический dashboard, который:

-упрощает анализ данных;
-делает dashboard более гибким;
-уменьшает количество дублирующихся графиков;
-повышает удобство работы конечного пользователя;
-позволяет быстро анализировать показатели в разных временных разрезах.
